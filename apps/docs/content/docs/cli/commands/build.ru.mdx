# ambrosia build

Читает файл `registry.json` (манифест), читает исходные файлы каждого пака и генерирует распространяемые JSON-файлы для хостинга registry. Аналог `shadcn build`.

## Использование

```bash
ambrosia build
```

## Что делает

<Steps>

### Читает `registry.json`

Манифест описывает все паки registry с указанием путей к файлам:

```json title="registry.json"
{
  "$schema": "https://registry.ambrosia.dev/schema/registry.json",
  "name": "my-registry",
  "items": [
    {
      "name": "jwt-auth",
      "version": "0.2.1",
      "description": "JWT аутентификация с guards и RBAC",
      "author": "community",
      "tags": ["jwt", "auth", "guard"],
      "category": "auth",
      "files": [
        { "path": "packs/jwt-auth/index.ts", "type": "index" },
        { "path": "packs/jwt-auth/jwt-auth.pack.ts", "type": "pack" },
        { "path": "packs/jwt-auth/jwt-auth.service.ts", "type": "service" },
        { "path": "packs/jwt-auth/jwt-auth.guard.ts", "type": "guard" },
        { "path": "packs/jwt-auth/types.ts", "type": "types" }
      ],
      "dependencies": { "jose": "^5.0.0" },
      "devDependencies": {},
      "registryDependencies": [],
      "ambrosiaDependencies": ["@ambrosia/core", "@ambrosia/http"]
    }
  ]
}
```

<Callout type="info">
  В `files[].path` указываются **пути к исходным файлам** относительно корня проекта. Поле `content` отсутствует — содержимое читается при сборке.
</Callout>

### Читает исходные файлы

Для каждого пака CLI читает все файлы по путям из `files[]` и встраивает их содержимое как строки.

### Генерирует JSON

Результат записывается в `dist/registry/`:

```
dist/registry/
├── index.json          ← список паков (без содержимого файлов)
└── jwt-auth.json       ← полный RegistryItem с content
```

</Steps>

## Формат registry.json (манифест)

```typescript
interface RegistryManifest {
  $schema?: string;
  name: string;           // Имя registry
  items: ManifestItem[];  // Паки для публикации
}

interface ManifestItem {
  name: string;
  version: string;
  description: string;
  author: string;
  tags: string[];
  category: PackCategory;
  files: { path: string; type: RegistryFileType }[];
  dependencies: Record<string, string>;
  devDependencies: Record<string, string>;
  registryDependencies: string[];
  ambrosiaDependencies: string[];
}
```

Ключевое отличие от `RegistryItem`: в манифесте `files` содержат только `path` и `type` — без `content`. Содержимое читается из файлов при выполнении `ambrosia build`.

## Структура registry-проекта

```
my-registry/
├── packs/
│   ├── jwt-auth/
│   │   ├── index.ts
│   │   ├── jwt-auth.pack.ts
│   │   ├── jwt-auth.service.ts
│   │   ├── jwt-auth.guard.ts
│   │   └── types.ts
│   └── redis-cache/
│       ├── index.ts
│       ├── redis-cache.pack.ts
│       ├── redis-cache.service.ts
│       └── types.ts
├── registry.json               ← манифест
└── dist/registry/              ← генерируется ambrosia build
    ├── index.json
    ├── jwt-auth.json
    └── redis-cache.json
```

## Ошибки

### Нет registry.json

```
✗  No registry.json found in current directory.
```

Создайте `registry.json` с описанием ваших паков.

### Файл не найден

```
✗  File not found: packs/jwt-auth/missing-file.ts
```

Путь в `files[].path` указывает на несуществующий файл. Проверьте путь.
