# ambrosia add

Downloads a pack from the registry and injects its source code into the project. Automatically resolves dependencies between packs and installs npm packages.

## Usage

```bash
ambrosia add <pack...> [--registry <name>]
```

## Arguments

| Argument | Description |
|---|---|
| `<pack...>` | One or more pack names to install |
| `--registry <name>` | Registry name from `ambrosia.json` (default: `"default"`) |

## Examples

```bash
# Single pack
ambrosia add jwt-auth

# Multiple packs
ambrosia add jwt-auth redis-cache config-pack

# From a specific registry
ambrosia add internal-auth --registry corp
```

## What It Does

### 1. Reading Configuration

The CLI reads `ambrosia.json` from the current directory. If the file is not found:

```
έεΩ  No ambrosia.json found. Run "ambrosia init" to create one.
```

### 2. Resolving Dependencies

The CLI fetches each requested pack from the registry and recursively downloads `registryDependencies`:

```
$ ambrosia add rbac-pack

έΩΗ  ambrosia add
έΦΓ
έΩΤ  Resolving dependencies...
έΩΘ  Resolved 2 pack(s)           έΗΡ rbac-pack + jwt-auth (dependency)
```

Resolution works depth-first έΑΦ dependencies are installed before dependent packs. Deduplication ensures each pack is installed only once.

### 3. Conflict Check

If a pack is already installed, the CLI asks:

```
έΩΗ  Pack(s) already installed: jwt-auth. Overwrite?
έΦΓ  έΩΜ Yes / έΩΠ No
```

Choosing "Overwrite" will replace the files. Your local changes will be lost.

### 4. Writing Files

Files from `files[]` are written to `{packsDir}/{packName}/`:

```
έΩΤ  Installing packs...
έΩΘ  Installed 2 pack(s)
```

Result:

```

έΦΓ   έΦεέΦΑέΦΑ jwt-auth.pack.ts
```

### 5. Installing npm Dependencies

The CLI collects all `dependencies`, `devDependencies`, and `ambrosiaDependencies` from the installed packs and runs:

```bash

```

```

```

## Using with a Custom Registry

```bash
# From a corporate registry
ambrosia add internal-auth --registry corp
```

The `"corp"` registry must be defined in `ambrosia.json`:

```json
{
  "registries": {
    "default": "https://registry.ambrosia.dev",
    "corp": "https://packs.mycompany.com"
  }
}
```

## Errors

### Pack Not Found

```
έεΩ  Pack "unknown-pack" not found in registry at https://registry.ambrosia.dev
```

Verify the pack name and registry availability.

### Circular Dependency

```
έεΩ  Circular registry dependency detected: pack-a έΗΤ pack-b έΗΤ pack-a
```

Packs cannot depend on each other cyclically. Use DI-level `lazyImports` for runtime circular dependencies.

### Registry Unavailable

```
έεΩ  Failed to fetch pack "jwt-auth": 500
```

Check your network connection and registry URL availability.

### Unknown Registry

```
έεΩ  Unknown registry "unknown". Available: default, corp
```

Add the registry to `ambrosia.json` under `registries`.
