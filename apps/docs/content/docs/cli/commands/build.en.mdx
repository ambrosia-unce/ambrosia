# ambrosia build

Reads the `registry.json` manifest, reads the source files for each pack, and generates distributable JSON files for hosting a registry. Similar to `shadcn build`.

## Usage

```bash
ambrosia build
```

## What It Does

<Steps>

### Reads `registry.json`

The manifest describes all registry packs with file paths:

```json title="registry.json"
{
  "$schema": "https://registry.ambrosia.dev/schema/registry.json",
  "name": "my-registry",
  "items": [
    {
      "name": "jwt-auth",
      "version": "0.2.1",
      "description": "JWT authentication with guards and RBAC",
      "author": "community",
      "tags": ["jwt", "auth", "guard"],
      "category": "auth",
      "files": [
        { "path": "packs/jwt-auth/index.ts", "type": "index" },
        { "path": "packs/jwt-auth/jwt-auth.pack.ts", "type": "pack" },
        { "path": "packs/jwt-auth/jwt-auth.service.ts", "type": "service" },
        { "path": "packs/jwt-auth/jwt-auth.guard.ts", "type": "guard" },
        { "path": "packs/jwt-auth/types.ts", "type": "types" }
      ],
      "dependencies": { "jose": "^5.0.0" },
      "devDependencies": {},
      "registryDependencies": [],
      "ambrosiaDependencies": ["@ambrosia/core", "@ambrosia/http"]
    }
  ]
}
```

<Callout type="info">
  The `files[].path` entries are **paths to source files** relative to the project root. The `content` field is absent — file contents are read during the build.
</Callout>

### Reads Source Files

For each pack, the CLI reads all files at the paths specified in `files[]` and embeds their contents as strings.

### Generates JSON

The output is written to `dist/registry/`:

```
dist/registry/
├── index.json          ← pack listing (without file contents)
└── jwt-auth.json       ← full RegistryItem with content
```

</Steps>

## registry.json Format (manifest)

```typescript
interface RegistryManifest {
  $schema?: string;
  name: string;           // Registry name
  items: ManifestItem[];  // Packs to publish
}

interface ManifestItem {
  name: string;
  version: string;
  description: string;
  author: string;
  tags: string[];
  category: PackCategory;
  files: { path: string; type: RegistryFileType }[];
  dependencies: Record<string, string>;
  devDependencies: Record<string, string>;
  registryDependencies: string[];
  ambrosiaDependencies: string[];
}
```

The key difference from `RegistryItem`: in the manifest, `files` contain only `path` and `type` — without `content`. Contents are read from files when `ambrosia build` runs.

## Registry Project Structure

```
my-registry/
├── packs/
│   ├── jwt-auth/
│   │   ├── index.ts
│   │   ├── jwt-auth.pack.ts
│   │   ├── jwt-auth.service.ts
│   │   ├── jwt-auth.guard.ts
│   │   └── types.ts
│   └── redis-cache/
│       ├── index.ts
│       ├── redis-cache.pack.ts
│       ├── redis-cache.service.ts
│       └── types.ts
├── registry.json               ← manifest
└── dist/registry/              ← generated by ambrosia build
    ├── index.json
    ├── jwt-auth.json
    └── redis-cache.json
```

## Errors

### No registry.json

```
✗  No registry.json found in current directory.
```

Create a `registry.json` with your pack descriptions.

### File Not Found

```
✗  File not found: packs/jwt-auth/missing-file.ts
```

The path in `files[].path` points to a non-existent file. Check the path.
