# ambrosia generate

Генерирует ресурсы для Ambrosia. Доступен короткий алиас `g`.

## Использование

```bash
ambrosia generate <type> <name>
ambrosia g <type> <name>
```

## Доступные генераторы

| Тип | Описание |
|---|---|
| `pack` | Создаёт pack (исходные файлы) внутри текущего проекта |

## ambrosia generate pack

Создаёт исходные файлы пака в директории `{packsDir}/{name}/` текущего проекта. Если `ambrosia.json` существует, использует `packsDir` из конфигурации. Иначе — `src/packs`.

```bash
ambrosia generate pack my-cache
ambrosia g pack my-cache
```

### Генерируемые файлы

```
src/packs/my-cache/
├── index.ts               ← barrel export
├── my-cache.pack.ts       ← PackDefinition с forRoot/forRootAsync
├── my-cache.service.ts    ← основной сервис
└── types.ts               ← интерфейс конфигурации + InjectionToken
```

<Callout type="info">
  Создаются **только исходные файлы** — никаких package.json, build.ts или README. Pack живёт как часть вашего проекта.
</Callout>

### Что включает каждый файл

**types.ts** — интерфейс конфигурации и `InjectionToken`:

```typescript title="src/packs/my-cache/types.ts"
import { InjectionToken } from '@ambrosia/core';

export interface MyCacheConfig {
  enabled: boolean;
}

export const MY_CACHE_CONFIG = new InjectionToken<MyCacheConfig>('MyCacheConfig');
```

**my-cache.pack.ts** — определение модуля с `forRoot` и `forRootAsync`:

```typescript title="src/packs/my-cache/my-cache.pack.ts"
export class MyCachePack {
  static forRoot(config: MyCacheConfig): PackDefinition {
    return {
      meta: { name: 'my-cache', version: '0.0.1' },
      providers: [
        { token: MY_CACHE_CONFIG, useValue: config },
        MyCacheService,
      ],
      exports: [MyCacheService],
    };
  }

  static forRootAsync(options: AsyncPackOptions<MyCacheConfig>): PackDefinition {
    // Асинхронная конфигурация
  }
}
```

**my-cache.service.ts** — сервис-заглушка:

```typescript title="src/packs/my-cache/my-cache.service.ts"
@Injectable()
export class MyCacheService {
  constructor(@Inject(MY_CACHE_CONFIG) private config: MyCacheConfig) {}

  getConfig(): MyCacheConfig {
    return this.config;
  }
}
```

**index.ts** — re-export всех публичных API:

```typescript title="src/packs/my-cache/index.ts"
export { MyCachePack } from './my-cache.pack';
export { MyCacheService } from './my-cache.service';
export type { MyCacheConfig } from './types';
export { MY_CACHE_CONFIG } from './types';
```

```

│
◇  Creating pack "my-cache" in src/packs/my-cache/
│
◆  Created 4 files
│
│  Usage
│  import { MyCachePack } from "./src/packs/my-cache";
│
│  packs: [
│    MyCachePack.forRoot({
│      enabled: true,
│    }),
│  ]
│
└  Pack ready!
```

### Следующие шаги

1. Отредактируйте `src/packs/my-cache/my-cache.service.ts` — добавьте бизнес-логику
2. Подключите pack в вашем приложении
3. Если хотите распространять pack через registry — см. [Публикация](/docs/cli/registry/publishing)
