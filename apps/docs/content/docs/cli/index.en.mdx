# @ambrosia-unce/cli

**@ambrosia-unce/cli** is a command-line tool for Ambrosia that lets you create projects, generate resources, and add **ready-made modules (packs)** from a registry directly into your project's source code.

## Philosophy

Unlike traditional npm packages, the **Pack Registry** follows a **code distribution** model — similar to [shadcn/ui](https://ui.shadcn.com) for React components. When you add a pack, its source code is copied into your project. You **own the code** and can freely modify it.

```
npm package:    node_modules/@lib/auth  →  closed, updated via npm
ambrosia pack:  src/packs/jwt-auth/     →  your code, full control
```

## Quick Example

```bash
# Create a project
ambrosia new my-app

# Initialize pack registry
cd my-app
ambrosia init

# Add a pack
ambrosia add jwt-auth
```

After running `ambrosia add jwt-auth`, your project will contain:

```
my-app/
├── src/
│   ├── packs/
│   │   └── jwt-auth/
│   │       ├── index.ts
│   │       ├── jwt-auth.pack.ts
│   │       ├── jwt-auth.service.ts
│   │       ├── jwt-auth.guard.ts
│   │       └── types.ts
│   └── main.ts
├── ambrosia.json
└── package.json
```

Wire up the pack in your application:

```typescript title="src/main.ts"
import { HttpApplication } from '@ambrosia-unce/http';
import { ElysiaProvider } from '@ambrosia-unce/http-elysia';
import { JwtAuthPack } from './packs/jwt-auth';

const app = await HttpApplication.create({
  provider: new ElysiaProvider(),
  packs: [
    JwtAuthPack.forRoot({
      secret: process.env.JWT_SECRET!,
      expiresIn: '1h',
    }),
  ],
});

await app.listen(3000);
```
