# @ambrosia/core

**@ambrosia/core** is a universal high-performance DI container (Dependency Injection) for TypeScript and Bun runtime. It provides powerful dependency management with minimal overhead and maximum flexibility.

## Quick Example

```typescript
import { Injectable, Container } from '@ambrosia/core';

@Injectable()
class DatabaseService {
  connect() {
    console.log('Connected to database');
  }
}

@Injectable()
class UserService {
  constructor(private db: DatabaseService) {}

  getUser(id: string) {
    this.db.connect();
    return { id, name: 'John Doe' };
  }
}

// Create container
const container = new Container({ mode: 'production' });

// Automatically resolve dependencies
const userService = container.resolve(UserService);
userService.getUser('123');
```

## Why @ambrosia/core?

### Universal DI

Unlike many DI containers that are tightly coupled to HTTP frameworks, **@ambrosia/core** is a pure DI container. Use it for:

- **CLI applications** - Command-line tools with clean dependency management
- **Desktop applications** - Electron, Tauri apps with proper architecture
- **Microservices** - HTTP, gRPC, message queue consumers
- **Worker threads** - Background tasks with dependency injection
- **Testing** - Easy mocking and dependency substitution

### Optimized for Bun

Built specifically for Bun runtime with:

- **AsyncLogger** - Non-blocking logging with `Bun.write()` and `queueMicrotask()`
- **Fast startup** - Optimized provider lookup and metadata caching
- **Zero overhead** - Production mode with minimal runtime checks
- **Memory efficient** - Smart GC hints and cache management

### Production Ready

- **51/51 tests pass** - Full test coverage
- **Benchmark suite** - 127+ performance tests in 6 suites
- **Type safe** - Full TypeScript support in strict mode
- **Battle tested** - Used in production microservices
