# Installation

## Requirements

<Callout type="info">
  **@ambrosia-unce/config** requires **@ambrosia-unce/core** as a peer dependency. Make sure core is already installed in your project.
</Callout>

- Bun 1.3.6 or higher
- TypeScript 5.0 or higher
- @ambrosia-unce/core 0.1.0 or higher

## Installing the Package

```bash
bun add @ambrosia-unce/config
```

## tsconfig Setup

Make sure decorators are enabled in your `tsconfig.json`:

```json
{
  "compilerOptions": {
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  }
}
```

## Basic Setup

### 1. Create a Configuration Schema

```ts title="src/config/app.config.ts"
import { defineConfig } from '@ambrosia-unce/config';

export const AppConfig = defineConfig({
  port: { env: 'PORT', type: 'int', default: 3000 },
  host: { env: 'HOST', type: 'string', default: 'localhost' },
  debug: { env: 'DEBUG', type: 'bool', default: false },
  jwtSecret: { env: 'JWT_SECRET', required: true },
});
```

### 2. Register the Module in a Pack

```ts title="src/app.pack.ts"
import { definePack } from '@ambrosia-unce/core';
import { ConfigModule } from '@ambrosia-unce/config';
import { AppConfig } from './config/app.config.ts';

export const AppPack = definePack({
  imports: [
    ConfigModule.forRoot({
      schema: AppConfig,
      envPath: '.env',      // optional
      validate: true,        // default: true
    }),
  ],
});
```

### 3. Use ConfigService

```ts title="src/services/app.service.ts"
import { Injectable } from '@ambrosia-unce/core';
import { ConfigService } from '@ambrosia-unce/config';

@Injectable()
export class AppService {
  constructor(private config: ConfigService) {}

  getPort(): number {
    return this.config.get<number>('port')!;
  }

  getSecret(): string {
    return this.config.getOrThrow<string>('jwtSecret');
  }
}
```

### 4. Create an .env File

```bash title=".env"
PORT=8080
HOST=0.0.0.0
DEBUG=true
JWT_SECRET=my-super-secret-key
```

<Callout type="warn">
  If `envPath` is not specified, `@ambrosia-unce/config` reads variables directly from `process.env` / `Bun.env`. The `.env` file is only used if the path is explicitly provided via `envPath`.
</Callout>

## Async Configuration

For cases where configuration depends on other services (secret managers, remote configs):

```ts
import { ConfigModule } from '@ambrosia-unce/config';
import { definePack } from '@ambrosia-unce/core';

const AppPack = definePack({
  imports: [
    ConfigModule.forRootAsync({
      useFactory: async () => ({
        schema: AppConfig,
        envPath: '.env',
        validate: true,
      }),
    }),
  ],
});
```
