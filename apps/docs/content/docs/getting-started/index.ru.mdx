# Что такое Ambrosia?

**Ambrosia** -- это TypeScript-фреймворк, созданный для рантайма [Bun](https://bun.sh). Он сочетает мощный DI-контейнер с провайдеро-агностичным HTTP-слоем, давая вам архитектуру enterprise-уровня на скорости Bun.

## Зачем Ambrosia?

Большинство TypeScript-фреймворков для бэкенда заставляют выбирать между скоростью и архитектурой. Ambrosia дает и то, и другое:

- **Декоратор-ориентированный DI** -- знакомые паттерны для тех, кто работал с NestJS или Spring, но спроектированные для Bun с нуля
- **Провайдеро-агностичный HTTP** -- замените Elysia на любой другой HTTP-движок без изменения бизнес-логики
- **Система паков** -- модульная архитектура с инкапсуляцией, а не просто соглашения по папкам
- **Compile-time валидация** -- типовая валидация с нулевой стоимостью в рантайме через TypeScript-трансформер
- **Event-driven подход** -- встроенная шина событий для слабосвязанной коммуникации между сервисами

## Для кого этот фреймворк?

Ambrosia подойдет вам, если вы:

- Разрабатываете продуктовые API и микросервисы на TypeScript
- Хотите надежные архитектурные паттерны без лишнего бойлерплейта
- Нуждаетесь в настоящем внедрении зависимостей, а не просто в service locator
- Заботитесь о производительности и хотите использовать скорость Bun
- Предпочитаете декоратор-ориентированные API с полной типобезопасностью

## Архитектура

```
@ambrosia-unce/core          -- Самостоятельный DI-контейнер (без привязки к HTTP)
@ambrosia-unce/http          -- Провайдеро-агностичный HTTP-слой (контроллеры, гарды, пайпы, интерцепторы)
@ambrosia-unce/http-elysia   -- Адаптер Elysia.js для @ambrosia-unce/http
@ambrosia-unce/config        -- Типобезопасная конфигурация из окружения
@ambrosia-unce/events        -- Высокопроизводительная шина событий
@ambrosia-unce/validator     -- Compile-time валидация типов
@ambrosia-unce/cli           -- Генерация проектов и кода
```

DI-контейнер не зависит от HTTP -- его можно использовать в CLI-утилитах, воркерах и любых TypeScript-приложениях. HTTP-слой -- это отдельный пакет, который подключается к ядру через систему паков.

## Как это выглядит

Вот как выглядит типичный API на Ambrosia:

```ts
import { Injectable, definePack } from '@ambrosia-unce/core';
import { Controller, Http, Body, Param, Status } from '@ambrosia-unce/http';
import { HttpApplication } from '@ambrosia-unce/http';
import { ElysiaProvider } from '@ambrosia-unce/http-elysia';

@Injectable()
class PostsService {
  private posts = [{ id: '1', title: 'Hello Ambrosia' }];

  findAll() {
    return this.posts;
  }

  create(title: string) {
    const post = { id: crypto.randomUUID(), title };
    this.posts.push(post);
    return post;
  }
}

@Controller('/posts')
class PostsController {
  constructor(private posts: PostsService) {}

  @Http.Get()
  findAll() {
    return this.posts.findAll();
  }

  @Http.Post()
  @Status(201)
  create(@Body() body: { title: string }) {
    return this.posts.create(body.title);
  }
}

const AppPack = definePack({
  providers: [PostsService],
});

const app = await HttpApplication.create({
  provider: ElysiaProvider,
  packs: [AppPack],
  controllers: [PostsController],
});

await app.listen(3000);
```
