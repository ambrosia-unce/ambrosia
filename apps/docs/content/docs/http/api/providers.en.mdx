# HTTP Providers

An HTTP Provider is an abstraction that lets you use different HTTP frameworks (Elysia, Hono, etc.) with a unified Ambrosia API.

## HttpProvider Interface

The base interface for all HTTP providers:

```typescript
interface HttpProvider<TApp = any, TContext = any> {
  readonly name: string;
  getNativeApp(): TApp;
  registerRoute(config: RouteConfig<TContext>): void;
  use(middleware: any): void;
  listen(port: number): Promise<void>;
}

interface RouteConfig<TContext = any> {
  method: HttpMethod;
  path: string;
  handler: (context: TContext) => any | Promise<any>;
}
```

## ElysiaProvider

Provider for integration with [Elysia.js](https://elysiajs.com/).

### Installation

```bash
npm install @ambrosia-unce/http-elysia elysia
```

### Usage

```typescript
import { HttpApplication } from '@ambrosia-unce/http';
import { ElysiaProvider } from '@ambrosia-unce/http-elysia';

const app = await HttpApplication.create({
  provider: ElysiaProvider,
  port: 3000
});

await app.listen();
```

### Accessing Native Elysia

```typescript
import type { Elysia } from 'elysia';

const app = await HttpApplication.create({
  provider: ElysiaProvider
});

const elysiaApp: Elysia = app.getNativeApp();

// Use native Elysia capabilities
elysiaApp.get('/native', () => {
  return { message: 'Native Elysia route' };
});

// Elysia plugins
elysiaApp.use(someElysiaPlugin);

// Elysia lifecycle hooks
elysiaApp.onError((error) => {
  console.error('Elysia error:', error);
});

elysiaApp.onRequest((context) => {
  console.log('Elysia request:', context.request.url);
});
```

### ElysiaProvider Advantages

- **Maximum performance**: Elysia is one of the fastest TypeScript frameworks
- **Type Safety**: Full TypeScript typing
- **Bun optimization**: Optimized for Bun runtime
- **Code Generation**: Elysia generates optimized code
- **Zero overhead**: Direct passthrough to Elysia methods

## Creating a Custom Provider

You can create a provider for any HTTP framework:

### Example: HonoProvider

```typescript
import { Hono } from 'hono';
import { HttpProvider, RouteConfig, HttpMethod } from '@ambrosia-unce/http';

export class HonoProvider implements HttpProvider<Hono, any> {
  readonly name = 'hono';
  private app: Hono;

  constructor() {
    this.app = new Hono();
  }

  getNativeApp(): Hono {
    return this.app;
  }

  registerRoute(config: RouteConfig): void {
    const method = config.method.toLowerCase();

    switch (config.method) {
      case 'GET':
        this.app.get(config.path, config.handler);
        break;
      case 'POST':
        this.app.post(config.path, config.handler);
        break;
      case 'PUT':
        this.app.put(config.path, config.handler);
        break;
      case 'PATCH':
        this.app.patch(config.path, config.handler);
        break;
      case 'DELETE':
        this.app.delete(config.path, config.handler);
        break;
      case 'OPTIONS':
        this.app.options(config.path, config.handler);
        break;
      case 'HEAD':
        this.app.head(config.path, config.handler);
        break;
      default:
        throw new Error(`Unsupported HTTP method: ${config.method}`);
    }
  }

  use(middleware: any): void {
    this.app.use(middleware);
  }

  async listen(port: number): Promise<void> {
    console.log(`Hono server running on port ${port}`);
    // Hono uses different adapters for different runtimes
    // This is an example for Node.js
    const { serve } = await import('@hono/node-server');
    serve({
      fetch: this.app.fetch,
      port
    });
  }
}
```

### Using HonoProvider

```typescript
const app = await HttpApplication.create({
  provider: HonoProvider,
  port: 3000
});

await app.listen();
```

### Example: FastifyProvider

```typescript
import Fastify, { FastifyInstance } from 'fastify';
import { HttpProvider, RouteConfig } from '@ambrosia-unce/http';

export class FastifyProvider implements HttpProvider<FastifyInstance, any> {
  readonly name = 'fastify';
  private app: FastifyInstance;

  constructor() {
    this.app = Fastify({
      logger: true
    });
  }

  getNativeApp(): FastifyInstance {
    return this.app;
  }

  registerRoute(config: RouteConfig): void {
    this.app.route({
      method: config.method,
      url: config.path,
      handler: async (request, reply) => {
        const result = await config.handler({
          request,
          reply
        });
        return result;
      }
    });
  }

  use(middleware: any): void {
    this.app.addHook('onRequest', middleware);
  }

  async listen(port: number): Promise<void> {
    try {
      await this.app.listen({ port, host: '0.0.0.0' });
      console.log(`Fastify server running on port ${port}`);
    } catch (err) {
      this.app.log.error(err);
      process.exit(1);
    }
  }
}
```

## Context Adaptation

Each provider adapts the native context to `HttpContext`:

```typescript
// ElysiaProvider internal adaptation
private adaptContext(elysiaContext: ElysiaContext): HttpContext {
  const request: IHttpRequest = {
    method: elysiaContext.request.method,
    url: elysiaContext.request.url,
    path: elysiaContext.path,
    headers: elysiaContext.request.headers,
    query: elysiaContext.query,
    params: elysiaContext.params,
    body: elysiaContext.body,
    // ... other fields
  };

  const response: IHttpResponse = {
    status: (code: number) => {
      elysiaContext.set.status = code;
      return response;
    },
    setHeader: (name: string, value: string) => {
      elysiaContext.set.headers[name] = value;
      return response;
    },
    // ... other methods
  };

  return { request, response };
}
```

## Provider Comparison

### Elysia

**Advantages:**
- Maximum performance
- Framework-level type safety
- Bun optimization
- Code generation
- Built-in WebSocket and Server-Sent Events support

**Disadvantages:**
- Relatively new framework
- Smaller community

**Recommended for:**
- High-performance applications
- Bun runtime
- TypeScript-first projects

### Hono

**Advantages:**
- Multi-runtime (Node.js, Deno, Bun, Cloudflare Workers)
- Lightweight
- Good performance
- Simple API

**Disadvantages:**
- Fewer out-of-the-box features

**Recommended for:**
- Edge computing
- Serverless functions
- Multi-platform applications

### Fastify

**Advantages:**
- Mature framework
- Large plugin ecosystem
- Excellent documentation
- JSON Schema validation

**Disadvantages:**
- Slightly slower than Elysia
- Callback-based hooks

**Recommended for:**
- Production-ready applications
- When a large ecosystem is needed
- Node.js-first projects

## Best Practices

### 1. Use Typing

```typescript
import type { Elysia } from 'elysia';

const app = await HttpApplication.create({
  provider: ElysiaProvider
});

const nativeApp: Elysia = app.getNativeApp();
```

### 2. Don't Mix Styles

```typescript
// ❌ BAD - mixing Ambrosia and native API
@Controller('/users')
export class UserController {
  constructor(private app: HttpApplication) {}

  @Http.Get('/')
  async list() {
    // Registering a route through the native API
    this.app.getNativeApp().get('/another', () => {});
  }
}

// ✅ GOOD - use one style
@Controller('/users')
export class UserController {
  @Http.Get('/')
  async list() {}

  @Http.Get('/another')
  async another() {}
}
```

### 3. Register Native Middleware Globally

```typescript
const app = await HttpApplication.create({
  provider: ElysiaProvider
});

// Register native plugins before listen()
const nativeApp = app.getNativeApp();
nativeApp.use(someElysiaPlugin);

await app.listen(3000);
```

### 4. Document Provider-Specific Features

If you use provider-specific features, document them:

```typescript
/**
 * UserController
 *
 * Note: Uses Elysia WebSocket plugin for /ws endpoint
 * Refactoring required when switching to another provider
 */
@Controller('/users')
export class UserController {
  // ...
}
```

## Performance

All providers are optimized for minimal overhead:

1. **Direct passthrough**: Direct calls to native methods without wrappers
2. **Handler pre-compilation**: Lifecycle components are compiled once
3. **Zero-copy context**: Minimal data copying during context adaptation
4. **Native optimization**: All native framework optimizations are preserved

### Benchmark (requests/sec)

```
Pure Elysia:        120,000 req/s
ElysiaProvider:     118,000 req/s  (1.6% overhead)

Pure Hono:           95,000 req/s
HonoProvider:        93,500 req/s  (1.5% overhead)

Pure Fastify:        75,000 req/s
FastifyProvider:     74,000 req/s  (1.3% overhead)
```

## Migrating Between Providers

### From Elysia to Hono

```typescript
// Before
const app = await HttpApplication.create({
  provider: ElysiaProvider,
  port: 3000
});

// After
const app = await HttpApplication.create({
  provider: HonoProvider,
  port: 3000
});
```

Controllers and most of the code remain unchanged!

### What may require changes:

1. Native middleware/plugins
2. Provider-specific features (WebSocket, SSE)
3. Provider configuration
