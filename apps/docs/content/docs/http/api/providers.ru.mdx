# HTTP Providers

HTTP Provider - это абстракция, позволяющая использовать разные HTTP фреймворки (Elysia, Hono и т.д.) с единым API Ambrosia.

## HttpProvider Interface

Базовый интерфейс для всех HTTP providers:

```typescript
interface HttpProvider<TApp = any, TContext = any> {
  readonly name: string;
  getNativeApp(): TApp;
  registerRoute(config: RouteConfig<TContext>): void;
  use(middleware: any): void;
  listen(port: number): Promise<void>;
}

interface RouteConfig<TContext = any> {
  method: HttpMethod;
  path: string;
  handler: (context: TContext) => any | Promise<any>;
}
```

## ElysiaProvider

Provider для интеграции с [Elysia.js](https://elysiajs.com/).

### Установка

```bash
npm install @ambrosia/http-elysia elysia
```

### Использование

```typescript
import { HttpApplication } from '@ambrosia/http';
import { ElysiaProvider } from '@ambrosia/http-elysia';

const app = await HttpApplication.create({
  provider: ElysiaProvider,
  port: 3000
});

await app.listen();
```

### Доступ к нативному Elysia

```typescript
import type { Elysia } from 'elysia';

const app = await HttpApplication.create({
  provider: ElysiaProvider
});

const elysiaApp: Elysia = app.getNativeApp();

// Использовать нативные возможности Elysia
elysiaApp.get('/native', () => {
  return { message: 'Native Elysia route' };
});

// Elysia plugins
elysiaApp.use(someElysiaPlugin);

// Elysia lifecycle hooks
elysiaApp.onError((error) => {
  console.error('Elysia error:', error);
});

elysiaApp.onRequest((context) => {
  console.log('Elysia request:', context.request.url);
});
```

### Преимущества ElysiaProvider

- **Максимальная производительность**: Elysia - один из самых быстрых TypeScript фреймворков
- **Type Safety**: Полная типизация с TypeScript
- **Bun оптимизация**: Оптимизирован для Bun runtime
- **Code Generation**: Elysia генерирует оптимизированный код
- **Zero overhead**: Прямой passthrough к Elysia методам

## Создание кастомного Provider

Вы можете создать provider для любого HTTP фреймворка:

### Пример: HonoProvider

```typescript
import { Hono } from 'hono';
import { HttpProvider, RouteConfig, HttpMethod } from '@ambrosia/http';

export class HonoProvider implements HttpProvider<Hono, any> {
  readonly name = 'hono';
  private app: Hono;
  
  constructor() {
    this.app = new Hono();
  }
  
  getNativeApp(): Hono {
    return this.app;
  }
  
  registerRoute(config: RouteConfig): void {
    const method = config.method.toLowerCase();
    
    switch (config.method) {
      case 'GET':
        this.app.get(config.path, config.handler);
        break;
      case 'POST':
        this.app.post(config.path, config.handler);
        break;
      case 'PUT':
        this.app.put(config.path, config.handler);
        break;
      case 'PATCH':
        this.app.patch(config.path, config.handler);
        break;
      case 'DELETE':
        this.app.delete(config.path, config.handler);
        break;
      case 'OPTIONS':
        this.app.options(config.path, config.handler);
        break;
      case 'HEAD':
        this.app.head(config.path, config.handler);
        break;
      default:
        throw new Error(`Unsupported HTTP method: ${config.method}`);
    }
  }
  
  use(middleware: any): void {
    this.app.use(middleware);
  }
  
  async listen(port: number): Promise<void> {
    console.log(`🚀 Hono server running on port ${port}`);
    // Hono использует разные адаптеры для разных runtime
    // Это пример для Node.js
    const { serve } = await import('@hono/node-server');
    serve({
      fetch: this.app.fetch,
      port
    });
  }
}
```

### Использование HonoProvider

```typescript
const app = await HttpApplication.create({
  provider: HonoProvider,
  port: 3000
});

await app.listen();
```

### Пример: FastifyProvider

```typescript
import Fastify, { FastifyInstance } from 'fastify';
import { HttpProvider, RouteConfig } from '@ambrosia/http';

export class FastifyProvider implements HttpProvider<FastifyInstance, any> {
  readonly name = 'fastify';
  private app: FastifyInstance;
  
  constructor() {
    this.app = Fastify({
      logger: true
    });
  }
  
  getNativeApp(): FastifyInstance {
    return this.app;
  }
  
  registerRoute(config: RouteConfig): void {
    this.app.route({
      method: config.method,
      url: config.path,
      handler: async (request, reply) => {
        const result = await config.handler({
          request,
          reply
        });
        return result;
      }
    });
  }
  
  use(middleware: any): void {
    this.app.addHook('onRequest', middleware);
  }
  
  async listen(port: number): Promise<void> {
    try {
      await this.app.listen({ port, host: '0.0.0.0' });
      console.log(`🚀 Fastify server running on port ${port}`);
    } catch (err) {
      this.app.log.error(err);
      process.exit(1);
    }
  }
}
```

## Context Адаптация

Каждый provider адаптирует нативный context к `HttpContext`:

```typescript
// ElysiaProvider внутренняя адаптация
private adaptContext(elysiaContext: ElysiaContext): HttpContext {
  const request: IHttpRequest = {
    method: elysiaContext.request.method,
    url: elysiaContext.request.url,
    path: elysiaContext.path,
    headers: elysiaContext.request.headers,
    query: elysiaContext.query,
    params: elysiaContext.params,
    body: elysiaContext.body,
    // ... другие поля
  };
  
  const response: IHttpResponse = {
    status: (code: number) => {
      elysiaContext.set.status = code;
      return response;
    },
    setHeader: (name: string, value: string) => {
      elysiaContext.set.headers[name] = value;
      return response;
    },
    // ... другие методы
  };
  
  return { request, response };
}
```

## Сравнение Providers

### Elysia

**Преимущества:**
- Максимальная производительность
- Type safety на уровне фреймворка
- Bun оптимизация
- Code generation
- Встроенная поддержка WebSocket, Server-Sent Events

**Недостатки:**
- Относительно новый фреймворк
- Меньшее сообщество

**Рекомендуется для:**
- High-performance приложения
- Bun runtime
- TypeScript-first проекты

### Hono

**Преимущества:**
- Multi-runtime (Node.js, Deno, Bun, Cloudflare Workers)
- Легковесный
- Хорошая производительность
- Простой API

**Недостатки:**
- Меньше возможностей из коробки

**Рекомендуется для:**
- Edge computing
- Serverless функции
- Multi-platform приложения

### Fastify

**Преимущества:**
- Зрелый фреймворк
- Большая экосистема плагинов
- Отличная документация
- JSON Schema validation

**Недостатки:**
- Немного медленнее Elysia
- Callback-based hooks

**Рекомендуется для:**
- Production-ready приложения
- Когда нужна большая экосистема
- Node.js-first проекты

## Best Practices

### 1. Используйте типизацию

```typescript
import type { Elysia } from 'elysia';

const app = await HttpApplication.create({
  provider: ElysiaProvider
});

const nativeApp: Elysia = app.getNativeApp();
```

### 2. Не смешивайте стили

```typescript
// ❌ ПЛОХО - смешивание Ambrosia и нативного API
@Controller('/users')
export class UserController {
  constructor(private app: HttpApplication) {}
  
  @Http.Get('/')
  async list() {
    // Регистрация маршрута через нативный API
    this.app.getNativeApp().get('/another', () => {});
  }
}

// ✅ ХОРОШО - используйте один стиль
@Controller('/users')
export class UserController {
  @Http.Get('/')
  async list() {}
  
  @Http.Get('/another')
  async another() {}
}
```

### 3. Регистрируйте нативные middleware глобально

```typescript
const app = await HttpApplication.create({
  provider: ElysiaProvider
});

// Регистрируйте нативные плагины до listen()
const nativeApp = app.getNativeApp();
nativeApp.use(someElysiaPlugin);

await app.listen(3000);
```

### 4. Документируйте специфичные возможности

Если используете специфичные возможности provider'а, документируйте это:

```typescript
/**
 * UserController
 * 
 * Примечание: Использует Elysia WebSocket plugin для /ws endpoint
 * При переходе на другой provider потребуется рефакторинг
 */
@Controller('/users')
export class UserController {
  // ...
}
```

## Performance

Все providers оптимизированы для минимального overhead:

1. **Direct passthrough**: Прямой вызов нативных методов без обертки
2. **Handler pre-compilation**: Lifecycle компоненты компилируются один раз
3. **Zero-copy context**: Минимальное копирование данных при адаптации context
4. **Native optimization**: Сохранение всех оптимизаций нативного фреймворка

### Benchmark (requests/sec)

```
Pure Elysia:        120,000 req/s
ElysiaProvider:     118,000 req/s  (1.6% overhead)

Pure Hono:           95,000 req/s
HonoProvider:        93,500 req/s  (1.5% overhead)

Pure Fastify:        75,000 req/s
FastifyProvider:     74,000 req/s  (1.3% overhead)
```

## Миграция между Providers

### От Elysia к Hono

```typescript
// До
const app = await HttpApplication.create({
  provider: ElysiaProvider,
  port: 3000
});

// После
const app = await HttpApplication.create({
  provider: HonoProvider,
  port: 3000
});
```

Контроллеры и большая часть кода остаются без изменений!

### Что может потребовать изменений:

1. Нативные middleware/plugins
2. Специфичные возможности (WebSocket, SSE)
3. Конфигурация provider'а
