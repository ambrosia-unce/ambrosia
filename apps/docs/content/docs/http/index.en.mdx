# @ambrosia/http

**@ambrosia/http** is a modern high-performance HTTP framework for TypeScript with a powerful Dependency Injection system. Built on top of [@ambrosia/core](/docs/core) and provides an abstraction over popular HTTP libraries (Elysia.js, Hono).

## Lifecycle Architecture

```
Incoming Request
    |
Middleware (@UseMiddleware)
    |
Guards (@UseGuard)
    |
Interceptors (before)
    |
Pipes (validation/transform)
    |
Route Handler
    |
Interceptors (after)
    |
Exception Filters
    |
Response
```

## Quick Example

```typescript
import { HttpApplication, Controller, Http, Body, UseGuard, Status } from '@ambrosia/http';
import { ElysiaProvider } from '@ambrosia/http-elysia';
import { Type } from '@sinclair/typebox';

// DTO with TypeBox schema
class CreateUserDto {
  static schema = Type.Object({
    name: Type.String({ minLength: 1 }),
    email: Type.String({ format: 'email' }),
    age: Type.Number({ minimum: 0 })
  });

  name!: string;
  email!: string;
  age!: number;
}

// Controller with auto-registration
@Controller('/users')
@UseGuard(AuthGuard)
export class UserController {
  constructor(private userService: UserService) {}

  @Http.Get('/')
  async list() {
    return this.userService.findAll();
  }

  @Http.Post('/')
  @Status(201)
  async create(@Body() dto: CreateUserDto) {
    // dto is automatically validated via ValidationPipe
    return this.userService.create(dto);
  }
}

// Start the application
const app = await HttpApplication.create({
  provider: ElysiaProvider,
  globalGuards: [AuthGuard],
  globalPipes: [ValidationPipe],
  port: 3000
});

await app.listen();
```

## Why @ambrosia/http?

### Performance Without Compromise

- **Zero-overhead abstraction** - Direct passthrough to native provider
- **WeakMap caching** - All metadata operations are cached
- **Handler pre-compilation** - Lifecycle components compiled once
- **Parameter optimization** - Specialized resolvers with fixed-length arrays

### Familiar Architecture

If you're familiar with NestJS, you'll feel right at home:

- **Same decorators** - @Controller, @Get, @UseGuard, @Body
- **Same lifecycle** - Middleware -> Guards -> Interceptors -> Pipes -> Handler
- **Same patterns** - Guards, Interceptors, Pipes, Exception Filters
- **But lighter** - No modules, no complex configuration

### Provider Flexibility

One codebase - different HTTP libraries:

```typescript
// Elysia.js
const app = await HttpApplication.create({
  provider: ElysiaProvider,
});

// Hono (in the future)
const app = await HttpApplication.create({
  provider: HonoProvider,
});
```

## Core Components

### Decorators (26 total)

**Controller:**
- `@Controller(path)` - Marks a class as a controller

**HTTP Methods:**
- `Http.Get()`, `Http.Post()`, `Http.Put()`, `Http.Delete()`, `Http.Patch()`, `Http.Options()`, `Http.Head()`

**Lifecycle:**
- `@UseGuard()`, `@UseInterceptor()`, `@UsePipe()`, `@UseMiddleware()`, `@UseFilter()`

**Response:**
- `@Status(code)` - Sets the HTTP status

**Parameters (11 types):**
- `@Body()`, `@Query()`, `@Param()`, `@Headers()`, `@Header()`
- `@Req()`, `@Res()`, `@Session()`, `@Cookie()`, `@Ip()`
- `@UploadedFile()`, `@UploadedFiles()`

### Built-in Exceptions

- `BadRequestException` (400)
- `UnauthorizedException` (401)
- `ForbiddenException` (403)
- `NotFoundException` (404)
- `ConflictException` (409)
- `UnprocessableEntityException` (422)
- `InternalServerErrorException` (500)

### Built-in Pipes

- `ValidationPipe` - Automatic validation with TypeBox
- `ParseIntPipe` - Convert string to number
- `ParseBoolPipe` - Convert string to boolean
