# Паки в HTTP

`HttpPackDefinition` расширяет базовый `PackDefinition` из `@ambrosia/core` добавляя поддержку контроллеров — HTTP-специфичной концепции, которая не относится к DI-ядру.

## HttpPackDefinition

```typescript
import type { HttpPackDefinition } from "@ambrosia/http";

export const UserPack: HttpPackDefinition = {
  meta: { name: "user" },
  controllers: [UserController],
  providers: [UserService, UserRepository],
  exports: [UserService],
};
```

### Отличие от PackDefinition

| Поле | `PackDefinition` (core) | `HttpPackDefinition` (http) |
|------|--------------------------|------------------------------|
| `providers` | + | + |
| `imports` | + | + |
| `exports` | + | + |
| `meta` | + | + |
| `onInit` / `onDestroy` | + | + |
| `controllers` | - | + |

<Callout type="info">
Контроллеры — это концепция HTTP-слоя. Базовый `PackDefinition` в `@ambrosia/core` работает только с DI-провайдерами. Если ваш пак не содержит контроллеров (например, пак для базы данных), используйте обычный `PackDefinition`.
</Callout>

## Структура HTTP-приложения

Типичное приложение организовано в feature-паки, каждый из которых содержит свои контроллеры, сервисы и провайдеры:

```
src/
├── modules/
│   ├── health/
│   │   ├── health.controller.ts
│   │   ├── health.service.ts
│   │   └── health.pack.ts
│   ├── user/
│   │   ├── user.controller.ts
│   │   ├── user.service.ts
│   │   ├── user.repository.ts
│   │   └── user.pack.ts
│   └── auth/
│       ├── auth.controller.ts
│       ├── auth.service.ts
│       └── auth.pack.ts
└── main.ts
```

### Feature Pack

```typescript
// src/modules/user/user.pack.ts
import type { HttpPackDefinition } from "@ambrosia/http";
import { UserController } from "./user.controller.ts";
import { UserService } from "./user.service.ts";
import { UserRepository } from "./user.repository.ts";

export const UserPack: HttpPackDefinition = {
  meta: { name: "user", version: "1.0.0" },
  controllers: [UserController],
  providers: [UserService, UserRepository],
  exports: [UserService], // Другие паки могут использовать UserService
};
```

### Регистрация в приложении

```typescript
// src/main.ts
import { HttpApplication } from "@ambrosia/http";
import { ElysiaProvider } from "@ambrosia/http-elysia";
import { HealthPack } from "./modules/health/health.pack.ts";
import { UserPack } from "./modules/user/user.pack.ts";
import { AuthPack } from "./modules/auth/auth.pack.ts";

const app = await HttpApplication.create({
  provider: ElysiaProvider,
  packs: [
    HealthPack,
    UserPack,
    AuthPack,
  ],
});

await app.listen(3000);
```

## Контроллеры и паки

Контроллеры из паков автоматически:
1. Регистрируются в DI-контейнере
2. Сканируются на наличие маршрутов (`@Get`, `@Post`, и т.д.)
3. Маршруты регистрируются в HTTP-провайдере

### Источники контроллеров

`HttpApplication` собирает контроллеры из трёх источников (все дедуплицируются):

```typescript
const app = await HttpApplication.create({
  provider: ElysiaProvider,

  // 1. Контроллеры из паков
  packs: [UserPack, AuthPack],

  // 2. Явный список контроллеров
  controllers: [HealthController],

  // 3. ControllerRegistry (side-effect import для обратной совместимости)
});
```

### Исключение контроллеров

```typescript
const app = await HttpApplication.create({
  provider: ElysiaProvider,
  packs: [UserPack, AuthPack],
  excludeControllers: [DebugController], // Не регистрировать в production
});
```

## Конфигурируемые паки

Паттерн `forRoot` / `forRootAsync` работает так же, как в core, но с добавлением контроллеров:

```typescript
import type { HttpPackDefinition, AsyncPackOptions } from "@ambrosia/http";
import { createAsyncProvider } from "@ambrosia/core";

export class AuthPack {
  static forRoot(config: AuthConfig): HttpPackDefinition {
    return {
      meta: { name: "auth" },
      controllers: [AuthController, OAuthController],
      providers: [
        { token: AUTH_CONFIG, useValue: config },
        AuthService,
        TokenService,
      ],
      exports: [AuthService],
    };
  }

  static forRootAsync(options: AsyncPackOptions<AuthConfig>): HttpPackDefinition {
    return {
      meta: { name: "auth-async" },
      controllers: [AuthController, OAuthController],
      providers: [
        createAsyncProvider(AUTH_CONFIG, options),
        AuthService,
        TokenService,
      ],
      exports: [AuthService],
    };
  }
}
```

Использование:

```typescript
const app = await HttpApplication.create({
  provider: ElysiaProvider,
  packs: [
    AuthPack.forRoot({ secret: process.env.JWT_SECRET!, expiresIn: "24h" }),
    UserPack,
  ],
});
```

## Условная загрузка паков

Массив `packs` поддерживает falsy-значения:

```typescript
const app = await HttpApplication.create({
  provider: ElysiaProvider,
  packs: [
    HealthPack,
    UserPack,
    process.env.ENABLE_ADMIN === "true" && AdminPack,
    process.env.NODE_ENV === "development" && SwaggerPack,
    null,      // игнорируется
    undefined, // игнорируется
  ],
});
```

## Lifecycle и Shutdown

### Graceful Shutdown

`app.close()` выполняет cleanup в правильном порядке:

```typescript
const app = await HttpApplication.create({
  provider: ElysiaProvider,
  packs: [
    DatabasePack.forRoot(dbConfig),
    CachePack.forRoot(cacheConfig),
    UserPack,
  ],
});

await app.listen(3000);

// При получении сигнала завершения:
process.on("SIGTERM", async () => {
  await app.close();
  process.exit(0);
});
```

`app.close()` выполняет:

1. **Pack onDestroy** — хуки паков в обратном порядке (LIFO)
2. **Injectable onDestroy** — `onDestroy()` всех tracked экземпляров (LIFO)
3. **Очистка кешей** — все scope-кеши контейнера

```typescript
// Порядок при shutdown:
// 1. UserPack.onDestroy()    (если определён)
// 2. CachePack.onDestroy()   (LIFO)
// 3. DatabasePack.onDestroy()
// 4. CacheService.onDestroy()    (Injectable lifecycle)
// 5. DatabaseService.onDestroy()
// 6. Очистка контейнера
```

## Смешивание core и HTTP паков

Пак базы данных не содержит контроллеров — для него подходит обычный `PackDefinition`:

```typescript
import type { PackDefinition } from "@ambrosia/core";

// Core pack — без контроллеров
export const DatabasePack: PackDefinition = {
  meta: { name: "database" },
  providers: [DatabaseService, ConnectionPool],
  exports: [DatabaseService],
};
```

```typescript
import type { HttpPackDefinition } from "@ambrosia/http";

// HTTP pack — с контроллерами
export const UserPack: HttpPackDefinition = {
  meta: { name: "user" },
  controllers: [UserController],
  providers: [UserService],
  imports: [DatabasePack], // Core pack как зависимость
};
```

Оба типа паков работают в массиве `packs` у `HttpApplication` — `PackDefinition` автоматически совместим с `HttpPackDefinition` (поле `controllers` просто отсутствует).

## CLI генерация

```bash
ambrosia new my-app
```

CLI генерирует проект с готовой структурой feature-паков, включая `HttpPackDefinition`, контроллеры и сервисы.
