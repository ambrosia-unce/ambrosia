# @ambrosia/validator

Type-safe compile-time validation для Ambrosia framework, использующий Bun plugin system и TypeScript Compiler API.

## Особенности

- ✅ **Нулевой runtime overhead** - код валидации генерируется на этапе компиляции
- ✅ **Type-safe** - полный вывод типов TypeScript
- ✅ **Без декораторов** - только TypeScript типы
- ✅ **Без дублирования схем** - типы и есть схема
- ✅ **Branded types** - Email, UUID, PositiveInt и другие
- ✅ **JSDoc constraints** - @minLength, @pattern и т.д.
- ✅ **Bun native** - использует Bun plugin system

## Быстрый пример

```typescript
import { assert } from '@ambrosia/validator';
import type { Email } from '@ambrosia/validator/types';

interface CreateUserDto {
  name: string;
  email: Email;
  age: number;
}

// Валидация на этапе компиляции!
const user = assert<CreateUserDto>({
  name: "Alice",
  email: "alice@example.com",
  age: 25
});
```

## Как это работает

Plugin использует TypeScript Compiler API для:

1. Поиска всех вызовов `validate<T>()`, `assert<T>()`, `is<T>()`
2. Анализа типа `T` через type checker
3. Генерации оптимизированного inline кода валидации
4. Замены вызова функции на сгенерированный код

### До трансформации:

```typescript
const user = assert<User>(data);
```

### После трансформации:

```typescript
const user = ((data) => {
  if (typeof data !== "object" || data === null)
    throw new ValidationError("must be an object");
  if (typeof data.name !== "string")
    throw new ValidationError("name must be string");
  if (typeof data.email !== "string")
    throw new ValidationError("email must be string");
  if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(data.email))
    throw new ValidationError("email must be valid");
  if (typeof data.age !== "number")
    throw new ValidationError("age must be number");
  return data;
})(data);
```

## Установка

```bash
bun add @ambrosia/validator
```

Добавьте в `bunfig.toml`:

```toml
preload = ["@ambrosia/validator/preload"]
```
